    97‚Üí    97‚Üídef simulate_first_touch(
    98‚Üí    98‚Üí    window_df,
    99‚Üí    99‚Üí    entry_price: float,
   100‚Üí   100‚Üí    sl_pct: float,
   101‚Üí   101‚Üí    tp_pct: float,
   102‚Üí   102‚Üí    direction: str,
   103‚Üí   103‚Üí) -> bool:
   104‚Üí   104‚Üí    """V4.4 (P7-FaseC, v0.40.23): Simulate LONG/SHORT trade on intraperiod OHLC.
   105‚Üí   105‚Üí
   106‚Üí   106‚Üí    Returns True if TP is touched before SL within the window. Conservative
   107‚Üí   107‚Üí    on ties (if both SL and TP touched in same candle, SL wins).
   108‚Üí   108‚Üí
   109‚Üí   109‚Üí    v0.40.24 CONTRACT: `window_df` MUST be the candles AFTER entry, not the
   110‚Üí   110‚Üí    candles that produced the pattern. The entry_price MUST be the close of
   111‚Üí   111‚Üí    the LAST pattern candle (‚âà open of the first post-pattern candle). The
   112‚Üí   112‚Üí    v0.40.23 implementation in ppmt.py / profiles.py passed the pattern's
   113‚Üí   113‚Üí    own window_df, which made `won` a circular function of the pattern
   114‚Üí   114‚Üí    itself ‚Äî see ppmt.py:337-422 for the fix rationale.
   115‚Üí   115‚Üí
   116‚Üí   116‚Üí    Args:
   117‚Üí   117‚Üí        window_df: pandas DataFrame with 'high' and 'low' columns ‚Äî the
   118‚Üí   118‚Üí            POST-entry candles (typically PATTERN_LEN √ó WINDOW candles
   119‚Üí   119‚Üí            immediately after the pattern ends). Empty/short DataFrames
   120‚Üí   120‚Üí            are handled gracefully (timeout = loss).
   121‚Üí   121‚Üí        entry_price: entry price (close of the LAST pattern candle, i.e.
   122‚Üí   122‚Üí            the candle immediately before window_df.iloc[0]).
   123‚Üí   123‚Üí        sl_pct: stop loss distance in percent (positive number).
   124‚Üí   124‚Üí        tp_pct: take profit distance in percent (positive number).
   125‚Üí   125‚Üí        direction: 'LONG' or 'SHORT'.
   126‚Üí   126‚Üí
   127‚Üí   127‚Üí    Returns:
   128‚Üí   128‚Üí        True if TP hit first, False if SL hit first or neither hit (timeout).
   129‚Üí   129‚Üí        Timeout is treated as loss (no TP within window).
   130‚Üí   130‚Üí    """
   131‚Üí   131‚Üí    if direction == "LONG":
   132‚Üí   132‚Üí        sl_price = entry_price * (1.0 - sl_pct / 100.0)
   133‚Üí   133‚Üí        tp_price = entry_price * (1.0 + tp_pct / 100.0)
   134‚Üí   134‚Üí    else:
   135‚Üí   135‚Üí        sl_price = entry_price * (1.0 + sl_pct / 100.0)
   136‚Üí   136‚Üí        tp_price = entry_price * (1.0 - tp_pct / 100.0)
   137‚Üí   137‚Üí
   138‚Üí   138‚Üí    for i in range(len(window_df)):
   139‚Üí   139‚Üí        high = float(window_df["high"].iloc[i])
   140‚Üí   140‚Üí        low = float(window_df["low"].iloc[i])
   141‚Üí   141‚Üí        if direction == "LONG":
   142‚Üí   142‚Üí            sl_hit = low <= sl_price
   143‚Üí   143‚Üí            tp_hit = high >= tp_price
   144‚Üí   144‚Üí        else:
   145‚Üí   145‚Üí            sl_hit = high >= sl_price
   146‚Üí   146‚Üí            tp_hit = low <= tp_price
   147‚Üí   147‚Üí        if sl_hit and tp_hit:
   148‚Üí   148‚Üí            return False  # conservative: SL wins on tie
   149‚Üí   149‚Üí        if sl_hit:
   150‚Üí   150‚Üí            return False
   151‚Üí   151‚Üí        if tp_hit:
   152‚Üí   152‚Üí            return True
   153‚Üí   153‚Üí    return False  # timeout = loss

   156‚Üí   156‚Üídef compute_outcome_won(
   157‚Üí   157‚Üí    window_df,
   158‚Üí   158‚Üí    entry_price: float,
   159‚Üí   159‚Üí    move_pct: float,
   160‚Üí   160‚Üí    sl_pct: Optional[float] = None,
   161‚Üí   161‚Üí    tp_pct: Optional[float] = None,
   162‚Üí   162‚Üí    historical_count: int = 0,
   163‚Üí   163‚Üí) -> bool:
   164‚Üí   164‚Üí    """V4.4 (P7-FaseC, v0.40.23): Compute `won` flag using outcome SL/TP.
   165‚Üí   165‚Üí
   166‚Üí   166‚Üí    Convenience wrapper used by ppmt.py / profiles.py / validator.py callers
   167‚Üí   167‚Üí    that have `window_df` available but want to apply the maturity threshold
   168‚Üí   168‚Üí    automatically.
   169‚Üí   169‚Üí
   170‚Üí   170‚Üí    Logic:
   171‚Üí   171‚Üí      1. Direction is inferred from sign(move_pct):
   172‚Üí   172‚Üí         move_pct > 0 ‚Üí LONG candidate, move_pct < 0 ‚Üí SHORT candidate.
   173‚Üí   173‚Üí         move_pct == 0 ‚Üí return False (degenerate, won't be classified).
   174‚Üí   174‚Üí      2. If historical_count < HIST_COUNT_MATURE: use OUTCOME_FLOOR_SL_PCT
   175‚Üí   175‚Üí         and OUTCOME_FLOOR_TP_PCT (bootstrap floors).
   176‚Üí   176‚Üí      3. If historical_count >= HIST_COUNT_MATURE: use the provided sl_pct /
   177‚Üí   177‚Üí         tp_pct (node's actual max_dd / max_fav from accumulated metadata).
   178‚Üí   178‚Üí      4. Simulate first-touch via simulate_first_touch().
   179‚Üí   179‚Üí
   180‚Üí   180‚Üí    Args:
   181‚Üí   181‚Üí        window_df: DataFrame with 'high' and 'low' columns for the POST-ENTRY
   182‚Üí   182‚Üí            window (typically PATTERN_LEN √ó WINDOW candles immediately AFTER
   183‚Üí   183‚Üí            the pattern ends). v0.40.24 CONTRACT: do NOT pass the pattern's
   184‚Üí   184‚Üí            own window_df here ‚Äî that produces a circular, meaningless `won`.
   185‚Üí   185‚Üí        entry_price: entry price (close of the LAST pattern candle, i.e. the
   186‚Üí   186‚Üí            candle immediately before window_df.iloc[0]).
   187‚Üí   187‚Üí        move_pct: observed move_pct = (exit - entry) / entry √ó 100. Used only
   188‚Üí   188‚Üí            to infer direction via sign ‚Äî the simulation itself runs on the
   189‚Üí   189‚Üí            POST-entry candles, not the pattern window that produced move_pct.
   190‚Üí   190‚Üí        sl_pct: node's |max_drawdown_pct| √ó 1.5 (None for new nodes).
   191‚Üí   191‚Üí        tp_pct: node's max(|expected_move_pct|, max_favorable_pct) √ó 1.0.
   192‚Üí   192‚Üí        historical_count: node's observation count BEFORE this observation
   193‚Üí   193‚Üí            (0 for the first observation in a new node).
   194‚Üí   194‚Üí
   195‚Üí   195‚Üí    Returns:
   196‚Üí   196‚Üí        True if outcome-SL/TP classifies this as a win, False otherwise.
   197‚Üí   197‚Üí    """
   198‚Üí   198‚Üí    if move_pct == 0:
   199‚Üí   199‚Üí        return False
   200‚Üí   200‚Üí    direction = "LONG" if move_pct > 0 else "SHORT"
   201‚Üí   201‚Üí
   202‚Üí   202‚Üí    if historical_count < HIST_COUNT_MATURE:
   203‚Üí   203‚Üí        # Young node: use bootstrap floors (more conservative than 0.10%
   204‚Üí   204‚Üí        # paper_trader floor ‚Äî fewer false positives on noisy first obs).
   205‚Üí   205‚Üí        use_sl = OUTCOME_FLOOR_SL_PCT
   206‚Üí   206‚Üí        use_tp = OUTCOME_FLOOR_TP_PCT
   207‚Üí   207‚Üí    else:
   208‚Üí   208‚Üí        # Mature node: use real SL/TP from accumulated metadata.
   209‚Üí   209‚Üí        use_sl = sl_pct if sl_pct is not None and sl_pct > 0 else OUTCOME_FLOOR_SL_PCT
   210‚Üí   210‚Üí        use_tp = tp_pct if tp_pct is not None and tp_pct > 0 else OUTCOME_FLOOR_TP_PCT
   211‚Üí   211‚Üí
   212‚Üí   212‚Üí    return simulate_first_touch(window_df, entry_price, use_sl, use_tp, direction)

   215‚Üí   215‚Üídef compute_outcome_directional(
   216‚Üí   216‚Üí    post_pattern_df,
   217‚Üí   217‚Üí    entry_price: float,
   218‚Üí   218‚Üí    move_pct: float,
   219‚Üí   219‚Üí) -> bool:
   220‚Üí   220‚Üí    """V5.0 (TAREA 15, v0.54.0): Compute `won` flag from REAL directional movement.
   221‚Üí   221‚Üí
   222‚Üí   222‚Üí    This is the BUILD-TIME replacement for compute_outcome_won(). Instead of
   223‚Üí   223‚Üí    simulating first-touch SL/TP with artificial 0.15% thresholds (which
   224‚Üí   224‚Üí    measures micro-structure noise, not predictive power), this function
   225‚Üí   225‚Üí    asks a simple question:
   226‚Üí   226‚Üí
   227‚Üí   227‚Üí        "Did the price move in the direction predicted by the pattern?"
   228‚Üí   228‚Üí
   229‚Üí   229‚Üí    TAREA 14 AUDIT FINDING: With SL/TP=0.15%, the first-touch simulation
   230‚Üí   230‚Üí    produced WR=46% on DOGE 1m ‚Äî below random. The root cause: 0.15% is
   231‚Üí   231‚Üí    18-22x smaller than real candle ranges (¬±1-5%), so micro-dips trigger
   232‚Üí   232‚Üí    SL before the real directional move materializes. Pattern "cbc" had
   233‚Üí   233‚Üí    expected_move_pct=+0.35% (REAL directional signal) but win_rate=41%
   234‚Üí   234‚Üí    (SL=0.15% triggered by noise first).
   235‚Üí   235‚Üí
   236‚Üí   236‚Üí    NEW LOGIC:
   237‚Üí   237‚Üí      1. Direction is inferred from sign(move_pct) of the PATTERN window.
   238‚Üí   238‚Üí      2. real_move_pct = (post_pattern close[-1] - entry_price) / entry_price √ó 100
   239‚Üí   239‚Üí         This is the ACTUAL price movement over the W candles after the pattern.
   240‚Üí   240‚Üí      3. For LONG: won = (real_move_pct > DIRECTIONAL_MICRO_FLOOR)
   241‚Üí   241‚Üí         For SHORT: won = (real_move_pct < -DIRECTIONAL_MICRO_FLOOR)
   242‚Üí   242‚Üí      4. Micro-floor of 0.01% filters degenerate zero-move cases only.
   243‚Üí   243‚Üí
   244‚Üí   244‚Üí    WHY THIS IS BETTER:
   245‚Üí   245‚Üí      - Measures PREDICTIVE POWER: "Did the pattern's direction call work?"
   246‚Üí   246‚Üí      - No artificial SL/TP thresholds that are timeframe-dependent
   247‚Üí   247‚Üí      - No first-touch noise contamination from micro-structure
   248‚Üí   248‚Üí      - WR naturally reflects genuine signal: random = 50%, skilled > 50%
   249‚Üí   249‚Üí      - Works identically across 1m/5m/15m ‚Äî the move_pct scales with time
   250‚Üí   250‚Üí
   251‚Üí   251‚Üí    Args:
   252‚Üí   252‚Üí        post_pattern_df: DataFrame with 'close' column for the POST-PATTERN
   253‚Üí   253‚Üí            window (W candles immediately AFTER the pattern ends). MUST have
   254‚Üí   254‚Üí            at least 1 row, otherwise returns False (timeout = loss).
   255‚Üí   255‚Üí        entry_price: entry price (close of the LAST pattern candle, i.e. the
   256‚Üí   256‚Üí            candle immediately before post_pattern_df.iloc[0]).
   257‚Üí   257‚Üí        move_pct: observed move_pct from the PATTERN window. Used ONLY to
   258‚Üí   258‚Üí            infer direction via sign ‚Äî the actual outcome is measured from
   259‚Üí   259‚Üí            post_pattern_df's close prices.
   260‚Üí   260‚Üí
   261‚Üí   261‚Üí    Returns:
   262‚Üí   262‚Üí        True if the post-pattern price moved in the predicted direction
   263‚Üí   263‚Üí        (beyond the micro-floor), False otherwise.
   264‚Üí   264‚Üí    """
   265‚Üí   265‚Üí    if move_pct == 0:
   266‚Üí   266‚Üí        return False
   267‚Üí   267‚Üí
   268‚Üí   268‚Üí    if len(post_pattern_df) == 0:
   269‚Üí   269‚Üí        return False  # timeout = loss
   270‚Üí   270‚Üí
   271‚Üí   271‚Üí    direction = "LONG" if move_pct > 0 else "SHORT"
   272‚Üí   272‚Üí
   273‚Üí   273‚Üí    # Real movement: last close of the post-pattern window vs entry
   274‚Üí   274‚Üí    actual_close = float(post_pattern_df["close"].iloc[-1])
   275‚Üí   275‚Üí    real_move_pct = (actual_close - entry_price) / entry_price * 100.0
   276‚Üí   276‚Üí
   277‚Üí   277‚Üí    if direction == "LONG":
   278‚Üí   278‚Üí        return real_move_pct > DIRECTIONAL_MICRO_FLOOR
   279‚Üí   279‚Üí    else:  # SHORT
   280‚Üí   280‚Üí        return real_move_pct < -DIRECTIONAL_MICRO_FLOOR
   281‚Üí   281‚Üí
   282‚Üí   282‚Üí
   283‚Üí   283‚Üí@dataclass
   284‚Üí   284‚Üíclass DirectionStats:
   285‚Üí   285‚Üí    """
   286‚Üí   286‚Üí    V4.3 (FIX-A): Per-direction statistics for a node's observation history.
   287‚Üí   287‚Üí
   288‚Üí   288‚Üí    Tracks win_rate, expected_move, and count separately for LONG and SHORT
   289‚Üí   289‚Üí    observations, enabling the trading engine to make direction-aware decisions

   595‚Üí   595‚Üí    - 'dependent': Relies on parent/ancestor metadata. Low count,
   596‚Üí   596‚Üí      inherited regime info. Confidence is scaled down by a factor
   597‚Üí   597‚Üí      based on the ratio of actual vs minimum observations.
   598‚Üí   598‚Üí    Classification is performed during propagate_metadata() and
   599‚Üí   599‚Üí    updated during Living Trie observations."""
   600‚Üí   600‚Üí
   601‚Üí   601‚Üí    min_independent_count: int = 10
   602‚Üí   602‚Üí    """Minimum historical_count for a node to be classified as independent.
   603‚Üí   603‚Üí    Nodes with count >= this threshold are 'independent'. Below it,
   604‚Üí   604‚Üí    they are 'dependent' and their metadata inherits from parents.
   605‚Üí   605‚Üí    10 is a reasonable default: below 10 observations, the node's
   606‚Üí   606‚Üí    statistics are too noisy to be reliable on their own."""
   607‚Üí   607‚Üí
   608‚Üí   608‚Üí    # === V4.2: Observation Freshness ===
   609‚Üí   609‚Üí    last_observation_time: float = 0.0
   610‚Üí   610‚Üí    """Timestamp (epoch seconds) of the most recent observation.
   611‚Üí   611‚Üí    V4.2: Enables observation freshness tracking ‚Äî patterns that
   612‚Üí   612‚Üí    haven't been observed recently can be deprioritized. This is
   613‚Üí   613‚Üí    critical for the Living Trie: as market conditions change,
   614‚Üí   614‚Üí    old patterns become less relevant. A pattern observed 5000
   615‚Üí   615‚Üí    candles ago should carry less weight than one observed 50 candles
   616‚Üí   616‚Üí    ago. The freshness_decay property computes a multiplier [0, 1]
   617‚Üí   617‚Üí    based on how long since the last observation."""
   618‚Üí   618‚Üí
   619‚Üí   619‚Üí    observation_timespan: float = 0.0
   620‚Üí   620‚Üí    """Time span (in seconds) between the first and last observation.
   621‚Üí   621‚Üí    V4.2: Measures how spread out observations are. A pattern observed
   622‚Üí   622‚Üí    100 times in one day is less reliable than one observed 100 times
   623‚Üí   623‚Üí    over 30 days. Longer timespan = more robust pattern that works
   624‚Üí   624‚Üí    across different conditions. Short timespan = potentially overfit
   625‚Üí   625‚Üí    to a specific market condition."""
   626‚Üí   626‚Üí
   627‚Üí   627‚Üí    # === Computed Properties ===
   628‚Üí   628‚Üí
   629‚Üí   629‚Üí    @property
   630‚Üí   630‚Üí    def risk_reward_ratio(self) -> float:
   631‚Üí   631‚Üí        """Compute risk:reward ratio from expected move vs max drawdown."""
   632‚Üí   632‚Üí        if self.max_drawdown_pct == 0:
   633‚Üí   633‚Üí            return 0.0
   634‚Üí   634‚Üí        return abs(self.expected_move_pct / self.max_drawdown_pct)
   635‚Üí   635‚Üí
   636‚Üí   636‚Üí    @property
   637‚Üí   637‚Üí    def confidence(self) -> float:
   638‚Üí   638‚Üí        """
   639‚Üí   639‚Üí        Confidence score based on historical observations and win rate.
   640‚Üí   640‚Üí        More observations and higher win rate ‚Üí higher confidence.
   641‚Üí   641‚Üí        Uses Bayesian-inspired shrinking toward 0.5 for low counts.
   642‚Üí   642‚Üí
   643‚Üí   643‚Üí        V4: Dependent nodes have their confidence scaled down because
   644‚Üí   644‚Üí        their metadata is inherited/aggregated rather than directly
   645‚Üí   645‚Üí        observed. An independent node with 50 observations is more
   646‚Üí   646‚Üí        trustworthy than a dependent node with 3 observations whose
   647‚Üí   647‚Üí        statistics were propagated from children.
   648‚Üí   648‚Üí        """
   649‚Üí   649‚Üí        if self.historical_count == 0:
   650‚Üí   650‚Üí            return 0.0
   651‚Üí   651‚Üí        # Bayesian shrinkage: prior of 0.5 with strength of 10 observations
   652‚Üí   652‚Üí        prior_strength = 10.0
   653‚Üí   653‚Üí        adjusted_win_rate = (
   654‚Üí   654‚Üí            (self.win_rate * self.historical_count + 0.5 * prior_strength)
   655‚Üí   655‚Üí            / (self.historical_count + prior_strength)
   656‚Üí   656‚Üí        )
   657‚Üí   657‚Üí        # Scale by sqrt of log(count) for sample size bonus
   658‚Üí   658‚Üí        count_bonus = min(1.0, np.sqrt(np.log1p(self.historical_count) / np.log(1000)))
   659‚Üí   659‚Üí        base_confidence = adjusted_win_rate * count_bonus
   660‚Üí   660‚Üí
   661‚Üí   661‚Üí        # V4: Dependent node penalty
   662‚Üí   662‚Üí        # Dependent nodes have less reliable metadata, so we scale
   663‚Üí   663‚Üí        # their confidence down. The penalty is proportional to how
   664‚Üí   664‚Üí        # far they are from the minimum independent count.
   665‚Üí   665‚Üí        if self.node_type == "dependent" and self.historical_count > 0:
   666‚Üí   666‚Üí            dependency_ratio = min(
   667‚Üí   667‚Üí                1.0, self.historical_count / self.min_independent_count
   668‚Üí   668‚Üí            )
   669‚Üí   669‚Üí            # Scale between 0.5 (0 observations toward independent)
   670‚Üí   670‚Üí            # and 1.0 (at the threshold)
   671‚Üí   671‚Üí            dependency_penalty = 0.5 + 0.5 * dependency_ratio
   672‚Üí   672‚Üí            base_confidence *= dependency_penalty
   673‚Üí   673‚Üí
   674‚Üí   674‚Üí        return base_confidence
   675‚Üí   675‚Üí
   676‚Üí   676‚Üí    @property
   677‚Üí   677‚Üí    def expected_profit_ahead(self) -> float:
   678‚Üí   678‚Üí        """
   679‚Üí   679‚Üí        Expected profit percentage looking ahead from this block.
   680‚Üí   680‚Üí        Combines win_rate √ó expected_move to give a realistic expectation.
   681‚Üí   681‚Üí
   682‚Üí   682‚Üí        This is the key metric the Money Manager uses to decide allocation:
   683‚Üí   683‚Üí          - High expected_profit_ahead ‚Üí allocate more capital
   684‚Üí   684‚Üí          - Low/negative ‚Üí allocate less or skip
   685‚Üí   685‚Üí
   686‚Üí   686‚Üí        Formula: win_rate √ó expected_move_pct + (1 - win_rate) √ó max_drawdown_pct
   687‚Üí   687‚Üí        This gives the expected value including losses.
   688‚Üí   688‚Üí        """
   689‚Üí   689‚Üí        if self.historical_count == 0:
   690‚Üí   690‚Üí            return 0.0
   691‚Üí   691‚Üí        win_expectation = self.win_rate * self.expected_move_pct
   692‚Üí   692‚Üí        loss_expectation = (1.0 - self.win_rate) * self.max_drawdown_pct
   693‚Üí   693‚Üí        return win_expectation + loss_expectation
   694‚Üí   694‚Üí
   695‚Üí   695‚Üí    @property
   696‚Üí   696‚Üí    def probability_of_success(self) -> float:
   697‚Üí   697‚Üí        """
   698‚Üí   698‚Üí        Probability that this pattern succeeds (reaches expected move
   699‚Üí   699‚Üí        before hitting stop loss).
   700‚Üí   700‚Üí
   701‚Üí   701‚Üí        This is more nuanced than raw win_rate because it accounts
   702‚Üí   702‚Üí        for the Bayesian shrinkage and sample size. The Money Manager
   703‚Üí   703‚Üí        uses this as the primary signal for position sizing.
   704‚Üí   704‚Üí
   705‚Üí   705‚Üí        Returns the same Bayesian-adjusted win_rate used in confidence(),
   706‚Üí   706‚Üí        but without the count bonus ‚Äî just the pure probability estimate.
   707‚Üí   707‚Üí        """
   708‚Üí   708‚Üí        if self.historical_count == 0:
   709‚Üí   709‚Üí            return 0.0
   710‚Üí   710‚Üí        prior_strength = 10.0
   711‚Üí   711‚Üí        return (
   712‚Üí   712‚Üí            (self.win_rate * self.historical_count + 0.5 * prior_strength)
   713‚Üí   713‚Üí            / (self.historical_count + prior_strength)
   714‚Üí   714‚Üí        )
   715‚Üí   715‚Üí
   716‚Üí   716‚Üí    @property
   717‚Üí   717‚Üí    def sizing_signal(self) -> float:
   718‚Üí   718‚Üí        """
   719‚Üí   719‚Üí        Composite sizing signal for the Money Manager (0.0 to 2.0+).
   720‚Üí   720‚Üí
   721‚Üí   721‚Üí        This is the single number the Risk Manager reads to decide
   722‚Üí   722‚Üí        position size. It combines:
   723‚Üí   723‚Üí          - probability_of_success: How likely is this pattern to win?
   724‚Üí   724‚Üí          - expected_profit_ahead: How much do we expect to make?
   725‚Üí   725‚Üí          - risk_reward_ratio: Is the payoff worth the risk?
   726‚Üí   726‚Üí
   727‚Üí   727‚Üí        Mapping (used by RiskManager):
   728‚Üí   728‚Üí          sizing_signal >= 1.5  ‚Üí 2.0x base position (high conviction)
   729‚Üí   729‚Üí          sizing_signal 1.0-1.5 ‚Üí 1.0x base position (normal)
   730‚Üí   730‚Üí          sizing_signal 0.5-1.0 ‚Üí 0.5x base position (low conviction)
   731‚Üí   731‚Üí          sizing_signal < 0.5   ‚Üí 0.25x or reject (very low)
   732‚Üí   732‚Üí
   733‚Üí   733‚Üí        This creates the tight PPMT ‚Üí RiskManager integration where
   734‚Üí   734‚Üí        the Trie's metadata directly drives capital allocation.
   735‚Üí   735‚Üí        """
   736‚Üí   736‚Üí        if self.historical_count == 0:
   737‚Üí   737‚Üí            return 0.0
   738‚Üí   738‚Üí
   739‚Üí   739‚Üí        # Normalize components
   740‚Üí   740‚Üí        prob = self.probability_of_success  # 0-1
   741‚Üí   741‚Üí
   742‚Üí   742‚Üí        # Expected profit: normalize to 0-1 range
   743‚Üí   743‚Üí        # A 2% expected profit is already very good for crypto
   744‚Üí   744‚Üí        profit_score = min(abs(self.expected_profit_ahead) / 2.0, 1.0)
   745‚Üí   745‚Üí
   746‚Üí   746‚Üí        # Risk:reward: normalize (RR of 3+ is excellent)
   747‚Üí   747‚Üí        rr_score = min(self.risk_reward_ratio / 3.0, 1.0)
   748‚Üí   748‚Üí
   749‚Üí   749‚Üí        # Weighted composite: probability is most important
   750‚Üí   750‚Üí        signal = 0.4 * prob + 0.35 * profit_score + 0.25 * rr_score
   751‚Üí   751‚Üí
   752‚Üí   752‚Üí        # Scale to 0-2 range for the multiplier
   753‚Üí   753‚Üí        return signal * 2.0
   754‚Üí   754‚Üí
   755‚Üí   755‚Üí    @property
   756‚Üí   756‚Üí    def is_unknown_block_exit(self) -> bool:
   757‚Üí   757‚Üí        """
   758‚Üí   758‚Üí        Whether an unknown next block should trigger an exit.
   759‚Üí   759‚Üí        True when there are continuation_nodes defined but the observed
   760‚Üí   760‚Üí        block is NOT among them ‚Äî meaning pattern broke.
   761‚Üí   761‚Üí        """
   762‚Üí   762‚Üí        return len(self.continuation_nodes) > 0
   763‚Üí   763‚Üí
   764‚Üí   764‚Üí    @property
   765‚Üí   765‚Üí    def move_std(self) -> float:
   766‚Üí   766‚Üí        """
   767‚Üí   767‚Üí        Standard deviation of observed moves.
   768‚Üí   768‚Üí        Computed from move_variance using Welford's online algorithm.
   769‚Üí   769‚Üí        High std = unpredictable pattern = lower effective confidence.
   770‚Üí   770‚Üí        Low std = consistent pattern = higher effective confidence.
   771‚Üí   771‚Üí        Returns 0.0 if fewer than 2 observations (can't compute variance).
   772‚Üí   772‚Üí        """
   773‚Üí   773‚Üí        if self.historical_count < 2:
   774‚Üí   774‚Üí            return 0.0
   775‚Üí   775‚Üí        return float(np.sqrt(self.move_variance / (self.historical_count - 1)))
   776‚Üí   776‚Üí
   777‚Üí   777‚Üí    @property
   778‚Üí   778‚Üí    def move_coefficient_of_variation(self) -> float:
   779‚Üí   779‚Üí        """
   780‚Üí   780‚Üí        Coefficient of variation (CV) of observed moves.
   781‚Üí   781‚Üí        CV = std / |mean|. Normalized measure of dispersion.
   782‚Üí   782‚Üí        CV < 0.5 = tight clustering around the mean (reliable)
   783‚Üí   783‚Üí        CV 0.5-1.0 = moderate dispersion (acceptable)
   784‚Üí   784‚Üí        CV > 1.0 = high dispersion (unreliable ‚Äî move direction uncertain)
   785‚Üí   785‚Üí        Returns 0.0 if expected_move_pct is zero (can't normalize).
   786‚Üí   786‚Üí        """
   787‚Üí   787‚Üí        if abs(self.expected_move_pct) < 1e-10:
   788‚Üí   788‚Üí            return 0.0
   789‚Üí   789‚Üí        return self.move_std / abs(self.expected_move_pct)
   790‚Üí   790‚Üí
   791‚Üí   791‚Üí    @property
   792‚Üí   792‚Üí    def freshness_decay(self) -> float:
   793‚Üí   793‚Üí        """
   794‚Üí   794‚Üí        V4.2: Observation freshness multiplier based on time since last observation.
   795‚Üí   795‚Üí
   796‚Üí   796‚Üí        Returns a value in [0, 1] that decays as the last observation gets older.
   797‚Üí   797‚Üí        Uses exponential decay with a half-life of 7 days (604800 seconds).
   798‚Üí   798‚Üí
   799‚Üí   799‚Üí        - 0 days old ‚Üí 1.0 (fresh, fully trusted)
   800‚Üí   800‚Üí        - 7 days old ‚Üí 0.5 (half-weight)
   801‚Üí   801‚Üí        - 30 days old ‚Üí ~0.06 (nearly expired)
   802‚Üí   802‚Üí
   803‚Üí   803‚Üí        This prevents stale patterns from having the same influence as
   804‚Üí   804‚Üí        recently-observed ones. In fast-moving markets, patterns that
   805‚Üí   805‚Üí        haven't been seen in weeks may no longer be valid.
   806‚Üí   806‚Üí
   807‚Üí   807‚Üí        Returns 1.0 if last_observation_time is 0 (not tracked).
   808‚Üí   808‚Üí        """
   809‚Üí   809‚Üí        if self.last_observation_time <= 0:
   810‚Üí   810‚Üí            return 1.0  # No tracking info, assume fresh
   811‚Üí   811‚Üí        import time as _time
   812‚Üí   812‚Üí        age_seconds = _time.time() - self.last_observation_time
   813‚Üí   813‚Üí        if age_seconds <= 0:
   814‚Üí   814‚Üí            return 1.0  # Future timestamp or same second
   815‚Üí   815‚Üí        # Half-life of 7 days = 604800 seconds
   816‚Üí   816‚Üí        half_life = 604800.0
   817‚Üí   817‚Üí        return float(np.exp(-0.693 * age_seconds / half_life))  # ln(2) ‚âà 0.693
   818‚Üí   818‚Üí
   819‚Üí   819‚Üí    @property
   820‚Üí   820‚Üí    def observation_density(self) -> float:
   821‚Üí   821‚Üí        """
   822‚Üí   822‚Üí        V4.2: Observations per unit time (observations/day).
   823‚Üí   823‚Üí
   824‚Üí   824‚Üí        Measures how concentrated observations are. Low density = pattern
   825‚Üí   825‚Üí        observed occasionally over a long time (robust). High density =
   826‚Üí   826‚Üí        pattern observed many times in a short period (potentially overfit).
   827‚Üí   827‚Üí
   828‚Üí   828‚Üí        Returns 0.0 if no timespan data.
   829‚Üí   829‚Üí        """
   830‚Üí   830‚Üí        if self.observation_timespan <= 0 or self.historical_count <= 0:
   831‚Üí   831‚Üí            return 0.0
   832‚Üí   832‚Üí        days = self.observation_timespan / 86400.0
   833‚Üí   833‚Üí        if days < 0.01:  # Less than ~15 minutes
   834‚Üí   834‚Üí            return float(self.historical_count) / 0.01  # Cap at 100/day
   835‚Üí   835‚Üí        return self.historical_count / days
   836‚Üí   836‚Üí
   837‚Üí   837‚Üí    def regime_match_score(self, current_regime: str) -> float:
   838‚Üí   838‚Üí        """
   839‚Üí   839‚Üí        V4.1: Compute a confidence multiplier based on regime match.
   840‚Üí   840‚Üí
   841‚Üí   841‚Üí        If the current market regime matches the node's dominant regime,
   842‚Üí   842‚Üí        confidence is boosted (up to 1.2x). If the current regime is
   843‚Üí   843‚Üí        unfavorable for this pattern, confidence is penalized (down to 0.5x).
   844‚Üí   844‚Üí
   845‚Üí   845‚Üí        The scoring uses both regime_distribution (how often) AND
   846‚Üí   846‚Üí        regime_stats (how well it performed):
   847‚Üí   847‚Üí
   848‚Üí   848‚Üí        1. If current regime matches dominant_regime ‚Üí boost (1.0 to 1.2)
   849‚Üí   849‚Üí           Boost is proportional to how dominant the regime is.
   850‚Üí   850‚Üí        2. If current regime exists but is not dominant ‚Üí neutral (0.8 to 1.0)
   851‚Üí   851‚Üí           Scaled by the ratio of observations in that regime.
   852‚Üí   852‚Üí        3. If current regime has NO observations ‚Üí penalty (0.5 to 0.7)
   853‚Üí   853‚Üí           Unknown territory, reduce confidence.
   854‚Üí   854‚Üí        4. If regime_stats available, further adjust by regime-specific win_rate
   855‚Üí   855‚Üí           vs overall win_rate. A regime where the pattern underperforms
   856‚Üí   856‚Üí           gets an additional penalty.
   857‚Üí   857‚Üí
   858‚Üí   858‚Üí        For independent nodes (sufficient observations), the regime match
   859‚Üí   859‚Üí        has MORE impact because we have reliable per-regime data.
   860‚Üí   860‚Üí        For dependent nodes (few observations), we apply less adjustment
   861‚Üí   861‚Üí        because the per-regime data is unreliable.
   862‚Üí   862‚Üí
   863‚Üí   863‚Üí        Args:
   864‚Üí   864‚Üí            current_regime: Current market regime string
   865‚Üí   865‚Üí
   866‚Üí   866‚Üí        Returns:
   867‚Üí   867‚Üí            Multiplier in range [0.5, 1.2] to apply to confidence
   868‚Üí   868‚Üí        """
   869‚Üí   869‚Üí        if not current_regime:
   870‚Üí   870‚Üí            return 1.0  # No regime info available, neutral
   871‚Üí   871‚Üí
   872‚Üí   872‚Üí        if not self.regime_distribution:
   873‚Üí   873‚Üí            return 1.0  # No regime data on this node, neutral
   874‚Üí   874‚Üí
   875‚Üí   875‚Üí        total_obs = sum(self.regime_distribution.values())
   876‚Üí   876‚Üí        if total_obs == 0:
   877‚Üí   877‚Üí            return 1.0
   878‚Üí   878‚Üí
   879‚Üí   879‚Üí        current_count = self.regime_distribution.get(current_regime, 0)
   880‚Üí   880‚Üí        current_ratio = current_count / total_obs
   881‚Üí   881‚Üí
   882‚Üí   882‚Üí        # Determine adjustment based on whether current regime is known
   883‚Üí   883‚Üí        if current_count == 0:
   884‚Üí   884‚Üí            # Regime never observed for this pattern ‚Äî penalty
   885‚Üí   885‚Üí            # Less severe for independent nodes (more diverse data)
   886‚Üí   886‚Üí            base_mult = 0.7 if self.node_type == "independent" else 0.5
   887‚Üí   887‚Üí            return base_mult
   888‚Üí   888‚Üí
   889‚Üí   889‚Üí        # Regime has been observed ‚Äî compute base multiplier
   890‚Üí   890‚Üí        if current_regime == self.dominant_regime:
   891‚Üí   891‚Üí            # Current regime is the dominant one ‚Äî boost
   892‚Üí   892‚Üí            # Boost proportional to dominance (how concentrated)
   893‚Üí   893‚Üí            boost = 1.0 + 0.2 * current_ratio  # 1.0 to 1.2
   894‚Üí   894‚Üí        else:
   895‚Üí   895‚Üí            # Regime exists but not dominant ‚Äî neutral to slight penalty
   896‚Üí   896‚Üí            # Scale by how rare this regime is for this pattern

  1306‚Üí  1306‚Üí    def best_direction_p7(

  1308‚Üí  1308‚Üí        min_edge_pct: float = 0.10,
  1309‚Üí  1309‚Üí        alpha: float = 1.0,

  1334‚Üí  1334‚Üí            return None

  1345‚Üí  1345‚Üí        if sc == 0:

  1353‚Üí  1353‚Üí        Args:
