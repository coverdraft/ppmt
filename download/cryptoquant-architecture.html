CryptoQuant Terminal — Arquitectura del Sistema
Pipeline end-to-end: Fuentes de datos → Brain → Estrategias → Backtesting → Paper Trading → Ejecucion Real
1 FUENTES DE DATOS APIs Externas
1CoinGecko Precios, volumen, market cap
2DexScreener Pares DEX, liquidez, OHLCV
3Etherscan Transacciones on-chain
4DexPaprika Datos DEX alternativos
5SQD / Dune / Footprint Datos avanzados (cuando disponibles)
↓ ohlcv-pipeline.ts → PriceCandle DB + Token DB
2 BRAIN PIPELINE (12 pasos) Analisis autonomo 24/7
1Discover Tokens Escaneo multi-chain
2Enrich Liquidity DexScreener batch
3Fetch OHLCV Velas reales CoinGecko
4Generate Signals SM, Rug Pull, V-Shape, LT
5Operability Filter Score tras fees + slippage
6System Matching Token → TradingSystem
7Persist & Store DB: Token, DNA, Signals
8Feedback Loop Brier score, validacion
9Growth Tracking CompoundGrowthTracker DB
10Bayesian Update Actualizacion prior/posterior
11Decision Engine OPERATE / SKIP / WATCH / EXIT
12Loop Schedule Ciclo continuo con intervalo
↓ brainAnalysis JSON → TokenAnalysis → StrategyOptimizer
3 AI STRATEGY MANAGER (6 pasos) Pipeline principal del usuario
1Setup Capital $10K, allocation mode, risk level
2Scan Tokens DexScreener + DB → TokenCandidate[]
3Generate Strategies StrategyConfig[] por categoria + risk
4Backtest Loop BacktestingEngine → BacktestRun DB
5Rank Results Composite score: Sharpe 30% + PnL 25% + WinRate 20%
6Activate & Trade TradingSystem → Paper Trading → Hall of Fame
↓ Mejor estrategia → TradingSystem (isPaperTrading=true)
4 RISK MANAGEMENT Distribuido en multiples engines
1Capital Allocation 16 metodos: Kelly, Markowitz, Risk Parity...
2Pre-Trade Checks 9 verificaciones antes de orden
3Stop Loss / Take Profit SL → TP → Trailing → Time → MaxDD
4Drawdown Manager WARNING 10% → DANGER 15% → CRITICAL 25%
5Statistical Validation Wilson CI, Welch t-test, power analysis
6Walk-Forward Analysis WFE robustez: ROBUST / MARGINAL / OVERFIT
↓ Estrategia validada + Risk controls → Paper Trading Engine
5 PAPER TRADING ENGINE Simulacion en vivo con precios reales
1Start Session Capital, maxPositions, SL/TP config
2Brain Analysis Token → Operability → Decision
3Open Position Size, entry price, SL/TP/trailing
4Price Sync 30s DexScreener → actualiza PnL/MFE/MAE
5Auto-Exit SL hit / TP hit / Trailing / Timeout
6Persist Trades PaperTradingTrade DB, sobrevive restarts
↓ Estrategias ganadoras validadas → Ejecucion Real (FUTURO)
6 EJECUCION REAL PENDIENTE — Zona robusta multi-wallet
1Broker Connectors Jupiter, Raydium, Uniswap APIs
2Multi-Wallet Manager Key management, balance tracking
3Order Execution Market/Limit, slippage control

Evaluacion de Risk Management & Features Faltantes
Position Sizing (16 metodos)
Excelente. Kelly Modified, Markowitz, Risk Parity, Vol Targeting... Con fee/slippage post-processing. El mas completo del sistema.
Walk-Forward Analysis
Ya implementado. WFE robustez, rolling/anchored, clasificacion ROBUST/MARGINAL/OVERFIT. Profesional.
Backtesting Engine
Sharpe/Sortino/Calmar, anti-overfitting, MFE/MAE. Debilidad: entry signals son heuristicas, no usan el pipeline real.
Stop Loss / Take Profit
Funciona. Prioridad: SL > TP > Trailing > Time > MaxDD. Trailing stop simplificado (usa bar high, no intra-trade high).
Drawdown Management
3 umbrales (10%/15%/25%) con cambio de modo automatico. Falta: auto-close de posiciones cuando se breach el umbral.
getCurrentUserId() = null
BLOQUEANTE. Sin auth activa, retorna null → 401 en todas las APIs protegidas. Fix: retornar demo user ID de la DB.
Monte Carlo Simulator
NO EXISTE. Simulacion de escenarios aleatorios para confidence bounds en backtest. Critico para quant serio.
Portfolio VaR / CVaR
NO EXISTE. Value-at-Risk a nivel portfolio. El sistema trackea riesgo individual pero no agregacion portfolio.
Broker Connectors
NO EXISTE. Ejecucion automatica de ordenes. Dejado para el final (zona robusta multi-wallet). Jupiter/Raydium APIs.
Portfolio Manager Multi-cuenta
NO EXISTE. Multi-wallet management, balance tracking, P&L consolidado. Pendiente con la zona de ejecucion real.
Scan → Generate (desconectado)
Scan obtiene tokens reales pero Generate crea templates genericos. Fix: pasar tokens del scan al generador.
Stress Testing
NO EXISTE. Flash crash, liquidity crisis, correlated drawdowns. Importante para gestion de riesgo real.
