Ingesta de Datos (Data Sources)
Entrada primaria: APIs externas + almacenamiento OHLCV
Binance Client PRIMARY
OHLCV en TODAS las temporalidades + volumen real + conteo de trades.
1200 req/min. ~500+ tokens con pares USDT.
Output: PriceCandle{tf,OHLCV,volume,trades,source="binance"}
CoinGecko Client SECONDARY
OHLCV limitado (30m, 4h). Sin volumen nativo (requiere getMarketChart separado).
~30 req/min free tier. 5000+ tokens via getTopTokensPaginated.

DexScreener Client SUPPLEMENT
Precios en vivo para Paper Trading. Pares DEX + liquidez + volumen.
Usado en syncOpenPositionPrices() cada 30s.

DexPaprika Client FALLBACK
35 cadenas. OHLCV a nivel de pool DEX. Backup cuando Binance/CoinGecko fallan.
Output: PriceCandle{source="dexpaprika"}
Etherscan Client ON-CHAIN
Transacciones on-chain para wallet profiling. Histórico de trades por wallet.
Output: TraderTransaction{action,valueUsd,pnlUsd,...}

Brain Cycle Engine (Operación 24/7)
Ciclo continuo: SCAN → FILTER → MATCH → STORE → FEEDBACK → GROWTH
Brain Cycle Engine CORE
Ejecuta ciclos periódicos (default: 5 min). Pipeline por ciclo:
1. Scan tokens de DB (volume24h DESC)
2. Capital Strategy Decision (position sizing)
3. Batch Operability Scoring (fee-aware filter)
4. Brain Analysis (analyzeBatch) en tokens operables
5. Trading System Matching
6. Store Operability Snapshots
7. Estimate cycle PnL (fee-deducted)
8. Compound Growth Tracking
9. Feedback Loop (auto si configurado)
10. Auto-backfill OHLCV
Brain Scheduler SCHED
Programa ciclos con estado persistente (DB).
start() / stop() / getPreviousState().
Auto-resume en restart via triggerAutoStart().
Capital Strategy Manager CAPITAL
Decide cómo asignar capital por ciclo. Modos: ULTRA_CONSERVATIVE, CONCENTRATED, DUAL.
Fee-aware: ajusta expectedGain y operability thresholds.
Aprende de resultados previos via updateFromCycleResult().
Trading System Matcher MATCH
batchMatchSystems() — asigna el mejor TradingSystem a cada token.
suggestEvolutions() — sugiere evoluciones de sistemas.
Token DNA Recalculator PERIODIC
Recalcula DNA de tokens stale cada 6 ciclos (~30 min).
Actualiza botActivityScore, smartMoneyScore, whaleScore.
↓ CycleResult{topPicks, capitalStrategy, operabilityDistribution}
3
Decision Layer (SDE + TDE)
Estrategia-level (SDE) y Token-level (TDE) decisions
Strategy Decision Engine (SDE) CENTRAL
EL MOTOR CENTRAL. Pipeline de 7 pasos:
STEP 1: Vetos duros (5 checks: minTrades, maxRiskOfRuin, maxDD, minWFE, minWinRate+payoff)
STEP 2: Composite Scores (Robustness, Overfitting, Stability)
STEP 3: Signal Quality (STRONG|ADEQUATE|WEAK)
STEP 4: State + Capital Action mapping
STEP 5: Capital Recommendation (5 métodos)
STEP 6: Recommendations
STEP 7: Audit record (siempre persistido)
Input: SDEInput{backtest, monteCarlo, walkForward, operability, portfolioState}
Output: {state: ACTIVE|CONDITIONAL|PAUSED|REJECTED, capitalAction: INCREASE|MAINTAIN|REDUCE|EXIT}
◆ dataQuality=PLACEHOLDER? → max CONDITIONAL + MAINTAIN
◆ paperTrading DD>15%? → PAUSED + EXIT
Token Decision Engine (TDE) TOKEN
Decisiones a nivel TOKEN (no estrategia).
decide() — 10 pasos: lifecycle → operability → risk → system → confidence → persist.
Decisiones: OPERATE|SKIP|WATCH|EXIT|ADJUST.
Mapeo lifecycle → sistema: GROWTH→SMART_MONEY, FOMO→ADAPTIVE, etc.
Output: DecisionResult{decisionType, recommendedSystem, confidence, reasoning}
Regime Heuristic REGIME
5 regímenes: TRENDING_UP, TRENDING_DOWN, SIDEWAYS, HIGH_VOLATILITY, LOW_VOLATILITY.
Prioridad: HIGH_VOL > LOW_VOL > TRENDING > SIDEWAYS.
Usa MA(7) vs MA(25) + vol percentile. Mínimo 25 precios.
Output: RegimeAssessment{regime, confidence, volatilityPercentile}
⚠ GAP P0: SDE→PTE integración rota — PTE tiene lógica inline, NO usa SDE. El pipeline /api/capital-allocation/pipeline es el único punto que conecta ambos.
⚠ GAP P1: TDE↔SDE coexisten sin integración. TDE no alimenta al SDE, SDE no consulta al TDE.
↓ StrategyDecision{state, capitalAction, capitalRecommendation} + KillSwitchState

Risk Management & Capital Allocation
Kill switches, Risk Budget, Capital Allocation, Correlation
Kill Switch Service CRITICAL
7 niveles de protección (hard constraints):
1. Global Pause (manual)
2. Portfolio DD >20% → PAUSE_ALL
3. Strategy DD >30% → PAUSE_STRATEGY
4. Position loss >50% → CLOSE_POSITION
5. Token concentration >15% → REJECT
6. Chain concentration >50% → REJECT
7. Sector concentration >30% → REJECT
Hysteresis: recovery al 80% del umbral. Alert automática al trigger.
Risk Budget (DB) CONFIG
RiskBudgetConfig en DB con defaults + cache 1 min.
Perfiles: CONSERVATIVE, MODERATE, AGGRESSIVE.
Umbrales por perfil en SDE (VETO_THRESHOLDS_BY_PROFILE).
Capital Allocation Engine ALLOC
16 métodos implementados, 5 activos en v1:
KELLY_MODIFIED (half-Kelly, default)
RISK_PARITY (equal risk contribution)
VOLATILITY_TARGETING (inverse vol)
MAX_DRAWDOWN_CONTROL (scale down with DD)
EQUAL_WEIGHT (fallback)
Fee/slippage post-processing integrado.
SDE Selection: DD>10% → MAX_DD_CONTROL | vol>75 → VOL_TARGET | strategies>2 → RISK_PARITY | else → KELLY
Strategy Correlation Service CORR
Matriz de correlación entre estrategias.
getCurrentCorrelationMatrix() + wouldExceedCorrelationLimit().
Si correlación >40% → reducir asignación a la mitad.
Monte Carlo Simulator MC
simulate(tradesPnlPct, config) — N simulaciones (1-100K).
Output: equityPercentiles, drawdownPercentiles, riskOfRuin, probabilityOfProfit.
Risk Controls Verifier VERIFY
Verifica controles de riesgo configurados.
⚠ GAP P1: maxDailyVaR faltante en RiskBudget Prisma schema y servicio.
⚠ GAP P1: Alert escalation chain (INFO→WARNING→CRITICAL→AUTO_PAUSE) no implementada.
⚠ GAP P2: RiskControlsVerifier tiene lógica hardcoded, sin análisis real.
↓ {allowed, positionSizeUsd, targetPct, method} + KillSwitch{actionRequired}

Paper Trading Engine (Simulación en Tiempo Real)
Motor de trading con persistencia DB, precios live, kill switches
Paper Trading Engine EXEC
Ciclo continuo con 2 timers:
Scan Loop (60s default):
  1. monitorOpenPositions() — verificar salidas
  2. fetchTopTokens() de DB
  3. analyzeToken() via Brain
  4. Operability check (doble: brain + operability engine)
  5. Concentration clamping (token + chain limits)
  6. Kill switch check (canOpenPosition())
  7. Open position con fee + slippage
Price Sync Loop (30s):
  1. Batch fetch DexScreener
  2. Update HWM/LWM (trailing stops)
  3. Kill switch evaluation (portfolio + strategy + position)
  4. Auto-close positions que pierden >50%
Exit Conditions EXIT
Stop Loss, Take Profit, Trailing Stop, Time-based exit, Kill Switch exit.
◆ pnlPct ≤ -stopLossPct? CLOSE
◆ pnlPct ≥ takeProfitPct? CLOSE
◆ trailing stop hit? CLOSE
Position Tracking PERSIST
DB: PaperTradingSession, PaperTradingPosition, PaperTradingTrade.
Restauración automática de posiciones en restart.
Fee normalization: DB stores %, config stores fraction.
activateStrategy() API
Entrada manual desde AI Strategy Manager.
Concentration clamping + Kill Switch check + openPosition.
⚠ GAP P0: Auto-feedback loop (S1.14) — feedbackLoopEngine importado pero NUNCA llamado desde PTE.
↓ PaperTradeRecord{pnl, holdTime, mfe, mae} + DecisionLog{outcome, wasCorrect}
